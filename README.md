# Функциональные требования MVP
# 1. Создание и редактирование workflow (ноды / рёбра)

### Пользователь может:

- создать новый workflow;
- добавлять ноды на canvas;
- соединять ноды ребрами;
- удалять ноды и связи;
- редактировать параметры ноды (URL, данные, переменные, код и т.п.);
- перемещать ноды по canvas;
- сохранять workflow.
# **2. Триггеры workflow**

### Поддержка трёх типов триггеров:

### **2.1. Ручной запуск**
- кнопка "Run" в UI;
- возможность указать входные параметры;

### **2.2. Scheduler**
- каждые N секунд/минут/часов,
- по будильнику
### **2.3. Webhook**
- автоматическая генерация URL для приема POST-запросов;
- входной JSON можно дальше юзать

# **3. HTTP Node
### Должны поддерживаться:
- методы: GET, POST, PUT, DELETE
- URL;
- headers (key/value);
- body
- таймаут;
- статус, тело, ошибки.
# **4. Полный просмотр стейта выполнения**
- видеть список узлов, которые выполнялись;
- видеть входные данные ноды;
- видеть выходные данные ноды;
- видеть ошибки, если были;
# 5. Потоки данных
- filter
- map
- reduce
- foreach
- flatmap

# **6. Code Node (Python)

Пользователь может писать Python-код в ноде. Это должно быть безопасно (нет лишних доступов, нельзя бесконечный цикл и тд, ограничены ресурсы). Доступ к данным прошлых нод.

# **7. Параллельное исполнение нод**
если граф имеет ветвление (например, от одного узла идут 2+ исходящих ребра), то эти ветки должны выполняться параллельно.
